# 產品需求文件 (PRD) - 任務管理系統 (Task Management System)

## 1. 專案概述 (Project Overview)

### 1.1 產品目的
打造一個簡潔、直覺的任務管理系統，讓使用者可以有效地記錄、追蹤與整理待辦事項。本系統旨在降低認知負擔，協助使用者專注於當前的核心工作。

### 1.2 目標用戶
**需要管理日常工作的個人用戶**：包括學生、上班族、自由工作者等，期望透過數位工具來條理化日常任務、提升個人工作效率的人群。

---

## 2. 系統功能需求 (Functional Requirements)

### 2.1 新增任務 (Add Task)
- **情境描述**：用戶可以隨時輸入文字來創建一個新的待辦任務。
- **輸入**：提供一個文字輸入框，輸入任務名稱（必填）。
- **觸發條件**：點擊「新增」按鈕或在輸入框內按下 `Enter` 鍵。
- **預期行為**：
  - 任務即時加入到清單中。
  - 新增任務的預設狀態為「未完成 (Active)」。
  - 新增後，輸入框應自動清空，方便連續輸入。
- **邊界處理**：若輸入內容為空白或全為空白字元，應阻止新增，並可給予輕微的視覺提示（如輸入框震動或顯示提示文字）。

### 2.2 標記任務完成 (Mark Task as Done)
- **情境描述**：用戶完成某項任務後，可在系統中將其標記為已完成。
- **操作方式**：點擊任務項目旁邊的核取方塊 (Checkbox) 或是任務狀態按鈕。
- **預期行為**：
  - 任務狀態變更為「已完成 (Completed)」。
  - 視覺上應有明顯區隔（例如：文字加上刪除線、顏色變淡灰、核取方塊顯示打勾）。
  - 支援「反悔機制」：再次點擊已完成的任務，可將其狀態恢復為「未完成」。

### 2.3 刪除任務 (Delete Task)
- **情境描述**：用戶可以將輸入錯誤或不再需要的任務從系統中永久移除。
- **操作方式**：點擊單一任務項目旁邊的「刪除 (Delete)」按鈕或圖示（如垃圾桶）。
- **預期行為**：該任務從清單中即時移除，不再顯示。

### 2.4 顯示所有任務清單 (Display Task List)
- **情境描述**：系統首頁需明確展示目前的任務列表。
- **顯示資訊**：每個任務項目應包含「狀態（完成與否）」、「任務名稱」、「刪除按鈕」。
- **列表呈現**：清單應隨著任務的增加向下延伸，必要時提供滾動軸。
- **無任務狀態 (Empty State)**：當清單中沒有任何任務時，應顯示友善的提示文字（例如：「目前沒有任務，開始規劃你的一天吧！」），避免畫面過於空曠。

### 2.5 依完成狀態篩選 (Filter by Status)
- **情境描述**：當任務數量變多時，用戶需要過濾畫面上的任務，以便專注處理進行中的事項，或檢視已完成的成果。
- **篩選條件**提供以下三個標籤頁/按鈕切換：
  1. **全部 (All)**：顯示所有新增的任務（不論完成與否）。
  2. **進行中 (Active)**：僅顯示狀態為「未完成」的任務。
  3. **已完成 (Completed)**：僅顯示狀態為「已完成」的任務。
- **預期行為**：
  - 點擊篩選條件後，下方清單應立即切換顯示符合該條件的任務。
  - 當前選中的篩選條件應有明顯的視覺高亮（Active State），讓用戶知道目前處於哪個過濾視圖。

## 3. 非功能性需求 (Non-Functional Requirements)

### 3.1 使用者介面與體驗 (UI/UX)
- **響應式設計 (Responsive Web Design)**：介面需適應不同螢幕尺寸，確保在桌上型電腦、平板及手機上都能提供良好的操作體驗（特別是點擊區域的設計需符合手機觸控佈局）。
- **微動畫回饋 (Micro-animations)**：在新增任務、刪除任務、標記完成、以及切換篩選條件時，加入平滑的轉場動畫（如淡入淡出、滑動），提升整體的質感與流暢度。

### 3.2 資料儲存 (Data Persistence)
- **本地儲存**：作為前端應用，應利用瀏覽器的本地儲存機制（如 `localStorage`）存儲任務資料。這樣即使用戶重新整理頁面或關閉瀏覽器後再次開啟，先前的任務資料依舊會保留，不會輕易消失。
